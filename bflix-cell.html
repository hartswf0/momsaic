Operations Manual
BFLIX as Computational Celluloid
Operational Paradigm

Recognizable images can be assembled under severe constraints. BFLIX is not about painting reality directly. It is about constructing odd building blocks that force perception to finish the image.

Rule: Think in discrete blocks, trajectories, holds, and shifts. The blur belongs to the viewer, not the script.

Grid Constraints
  • Logical grid is 64 columns by 36 rows.
  • Origin is top-left: 0,0.
  • Values map to palette ids 0-7.
  • 80 columns remains a historical guide, but the compiler does not hard-stop on longer lines.
Command Set
  • CLR v clears the matrix to a value.
  • PNT x y w h v paints a block.
  • LIN x1 y1 x2 y2 v draws a line.
  • REC n records the current frame n times.
  • SHF dx dy n shifts the whole matrix over n frames.
